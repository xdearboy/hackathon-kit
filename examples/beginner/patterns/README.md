# Patterns

Короткие объяснения паттернов без лишней теории.

## Оглавление

- `adapter.md` — подружить разные интерфейсы
- `builder.md` — собирать объект по шагам
- `decorator.md` — добавить поведение через обёртку
- `facade.md` — один простой интерфейс поверх подсистем
- `factory.md` — создавать объект по типу
- `observer.md` — рассылать события подписчикам
- `singleton.md` — один общий объект на всё приложение
- `strategy.md` — подставлять алгоритм как объект

## Как использовать

Сначала прочитай идею, потом открой соответствующую задачу в `tasks/`.
